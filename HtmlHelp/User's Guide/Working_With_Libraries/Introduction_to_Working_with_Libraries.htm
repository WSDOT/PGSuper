Chapter 11 - Working with Libraries

The Library is one of the most powerful features in PGSuper. Libraries allow you to catalog and share commonly-used input data in a common location. In order for PGSuper to model real bridge structures and to perform thorough specification compliance checking, a lot of detailed information is required. In government organizations like WSDOT and TxDOT, much of the data is standardized. You will find data about standard concrete mixes, standard girder plans, standard connection details, and the like. While all this information is common, and well documented, it is tedious and error-prone to enter into a computer program every time you need to use it. Libraries enable data to be input once, and reused for many projects.

PGSuper Libraries

PGSuper has six different libraries. Each library is described below.

Concrete

This library contains standard definitions for concrete.

Connections

Connections describe the end regions of a precast girder, the end diaphragm, and how the dead load of the end diaphragm is applied to the system.

Girders

Entries in this library describe standard precast girders. Girders are described by not only their cross sectional dimensions, but by strand patterns, longitudinal mild steel, transverse mild steel, and harping point locations.

Traffic Barriers

Entries in this library describe the cross sectional dimensions of standard traffic barriers.

Project Criteria

 

Entries in this library describe project criteria such as: the version of the LRFD Specifications, allowable stress overrides, choices of computations methods for prestress loss, creep, live load distribution factors, and parameters for long girder stability analysis.

Vehicular Live Load

Entries in this library describe user defined live loads

Detailed information for each type of library can be found Appendix B.

Master Library Entries and Project Library Entries

The PGSuper library greatly simplifies user input, and it allows organizations to standardize PGSuper input data. Libraries allow standard information to be recorded and referenced by a user-defined name. For example, it can require hundreds of pieces of data to describe the section dimensions, strand data, reinforcing data, etc. for a parametric precast girder. Library entries allow you to define this data once and use it over and over again in different design projects. But, if necessary, you can also define library entries that are unique to your current project.

The PGSuper Library is comprised of two different types of entries: Master Library Entries and Project Library Entries. As you might guess, Master Library Entries are stored in the Master Library. The Master Library is typically stored out on a network so all the engineers within an organization can share it. This makes it easy for an entire team of bridge designers to work with the same information. Project Library Entries are specific to a PGSuper project and are stored locally in your Project File. You can create your own Project Library Entries by copying and modifying the Master Library Entries or by creating them from scratch.

Tip: Designate someone in your organization as the PGSuper Administrator. The PGSuper Administrator's job is to maintain the master library file and coordinate installations of new versions of PGSuper. This will help to insure that you are using an up-to-date version of PGSuper and, by maintaining a single library, improve quality in our organization. Refer to the Administrator's Guide for more information.
