Program Settings

Program Settings customize the functionality of PGSuper. Using the Programs Settings dialog, you can customize your user information, define template folders, and set the location of the master library file.

To access the Program Settings dialog:

  1. Close your PGSuper project if you have one open. Select File | Close (ALT,F,C)

  2. Select File | Program Settings (ALT,F,G). This will display the Program Settings dialog.

Libraries and Templates

You can customize the location of library and templates used by PGSuper. At installation, PGSuper provides some "generic" defaults to get you started. You can also use files that are published on the Internet, or create your own libraries and templates.

There are two options for the location of libraries and templates. Select the one that best fits your situation.

Use generic libraries and templates

If you are just trying out PGSuper, this is a good option. PGSuper will use the libraries and templates that were installed with the program. Note that these templates are based on WSDOT settings, but they are not automatically kept up to date over the Internet.

Also note that this gives the appearance that PGSuper support a very limited number of girder types. This is not the case. PGSuper supports a wide variety of parametric girder types.

Use libraries and templates published on your local file system, local area network, or the Internet

This is the typically the option you'll want to use if your organization is doing production work for a bridge owner. DOTs like WSDOT and TxDOT publish libraries and templates on the Internet that reflect their policies and practices. Also, you can configure PGSuper to share libraries and templates locally, or on a server in your office.

By selecting this option, PGSuper's libraries and templates are automatically kept up to date at the frequency set in the dialog.

PGSuper Catalog Servers

Agencies such as WSDOT and TxDOT can publish catalogs of libraries and project templates on the Internet, on your local computer's file system, or on your local area network. You can set up your own server to provide libraries and templates. Select the server that you would like to use. To manage servers, press the Add/Edit button. From here you can add, edit, and delete server definitions.

A catalog server is defined by a name and a location. The location must be an FTP, HTTP, or file server with anonymous access. An example of a PGSuper Catalog Server definition is:

Name

Location

WSDOT

ftp://ftp.wsdot.wa.gov/public/bridge/Software/PGSuper

User Information

On the User Information tab of the Program Settings Dialog, you can enter some information about yourself and where you work. This information is then transferred to the Project Properties data when you create a new PGSuper project.

Update Frequency

PGSuper periodically queries the catalog server to see if the library and template files have been changed since the last update. This setting tells PGSuper how often to check in with the catalog server.
