Program Settings

Program Settings customize the functionality of PGSuper. Using the Programs Settings dialog, you can customize your user information, define template folders, and set the location of the master library file.

To access the Program Settings dialog:

  1. Close your PGSuper project if you have one open. Select File | Close (ALT,F,C)

  2. Select File | Program Settings (ALT,F,G). This will display the Program Settings dialog.

Libraries and Templates

You can customize the library and templates used by PGSuper.

There are three options for the libraries and templates. Select the one that best fits your situation.

Use generic libraries and templates

If you are just trying out PGSuper, this is a good option. PGSuper will use the libraries and templates that were installed with the program. Note that these templates are based on WSDOT settings, but they are not automatically kept up to date over the Internet.

Also note that this gives the appearance that PGSuper only supports WSDOT girders. This is not the case. PGSuper supports a wide variety of parametric girder types.

Download and synchronize with published libraries and templates

This is the option you'll want to use if you are doing production work for a bridge owner. DOTs like WSDOT and TxDOT publish libraries and templates on the Internet that reflect their policies and practices. By selecting this option, PGSuper's libraries and templates are automatically kept up to date via the Internet.

Contact WSDOT if you would like to publish your own libraries and templates.

Use and synchronize with libraries and templates on your local network

This is the option you'll want to use if you want to share libraries and templates that are not published on the Internet.

PGSuper Catalog Server

Agencies such as WSDOT and TxDOT can publish catalogs of libraries and project templates on the Internet. Select the catalog server that you would like to use. To manage your catalog servers, press the Add/Edit button. From here you can add, edit, and delete server definitions.

A catalog server is defined by a name and a location. The location must be an FTP server with anonymous access. An example of a PGSuper Catalog Server definition is:

Name

Location

WSDOT

ftp://ftp.wsdot.wa.gov/public/bridge/Software/PGSuper

User Information

On the User Information tab of the Program Settings Dialog, you can enter some information about yourself and where you work. This information is then transferred to the Project Properties data when you create a new PGSuper project.

User Templates Folder

The user templates folder provides a secondary location where you can store PGSuper template files. This can be on your local system, or in a shared location in your local network. Store them in a folder and set the User Templates field to this folder. PGSuper will then offer the templates stored in this folder, and in folders directly below it, when creating new projects.

Note: See Chapter 11  of the PGSuper User Guide for more information on Project Templates.

Update Frequency

PGSuper periodically queries the catalog server to see if there are updated libraries or templates. This setting tells PGSuper how often to check in with the catalog server.

Note: See Chapter 2 and Chapter 3 of the PGSuper Administrator's Guide for more information on Shared Libraries and Shared Project Templates.
