Chapter 13 - Project Templates

Way back in Chapter 2, you learned how to create a new project using a Project Template. In this chapter you will learn how to create and manage your own project templates.

To review, a Project Template is a special project file that contains standard configurations for bridge geometry and components, appurtenances, loading, and design criteria. Templates speed the creation of new projects by setting up much of the input for you.

Tip: Why use Project Templates? A wise man once said:  "If I have seen a little further it is by standing on the shoulders of Giants." or, perhaps it was: "Why bother creating something new, when we can just plagiarize from others". Anyway, this is the idea behind Project Templates. They allow you to create standard projects for different common scenarios, so they can be used to jump-start new projects.

"Why shouldn't I just copy my old project files?", you might say. Project Templates are an improvement over just using input files in two ways:

  1. When you open a new file using a Project Template, the Project Properties are updated using your custom settings;

  2. PGSuper offers slick features for managing Project Templates and for sharing them among your workgroup including engineers scattered across the globe.

This chapter covers the creation and management of templates. For information regarding creating and managing Project Templates for an entire engineering organization, also see Chapter 3 of the PGSuper Administrator's Guide.
