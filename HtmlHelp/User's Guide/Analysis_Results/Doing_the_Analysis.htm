Doing the Analysis

Unlike most batch oriented programs, you do not have to tell PGSuper to Do The Analysis. By default, PGSuper continuously analyzes your bridge and keeps up to date the analysis results that are being displayed. To view analysis results, simply open the Analysis Results View and select a diagram, or create a report in the Report View. PGSuper will do whatever work is necessary to present you with the requested information.

This is part of the Bridge-centric user interface WSDOT has created. The basic idea is this. Keep the engineer focused on the engineering problem and not the details of the software. You are running PGSuper because you need to know some specific information about your structure. A Bridge-centric interface allows you to stay centered on your goal and forget about many of the details of the software.

Of course, this continuous processing only works well on fast computers solving relatively small problems. In the future, as our computing power increases, continuous processing will work better. The continuous processing is controlled by the AutoCalc mode. The next chapter covers AutoCalc in detail.

This chapter assumes AutoCalc mode is turned on.
