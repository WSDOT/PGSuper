Capacity/Demand (C/D) Ratios

C/D ratios are reported in several places in PGSuper (examples are stresses, moment capacity,...) The rules for reporting the ratios are consistent throughout, and are as follows:

  • C/D is only computed when the signs of Capacity and Demand are the same
  • C/D’s are reported using two decimal places in the form “(2.22)”
  • If C/D >= 10.0, report as “(10+)”
  • If Demand exceeds Capacity, this is always a FAIL. A tolerance is never applied  to make C/D pass. To avoid confusion and if needed, force reported C/D value to “(0.99) for fails outside of 2 significant figures. For example, if C/D=0.999999, report a FAIL with “(0.99)”
  • For the case of C=0.0 (e.g., zero allowable tension) do not report C/D, only report PASS/FAIL.
  • If D=0.0, and C!=0.0 report infinity.
  • For the case of sign reversals, report C/D as “(-)”. For example, C is positive moment capacity and D is a negative moment.
