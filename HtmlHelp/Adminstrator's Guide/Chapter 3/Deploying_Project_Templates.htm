Deploying Project Templates

After you create your project templates, they must be deployed to your users. There are two ways to deploy project templates. You can copy them locally onto every user's computer, or you can store them in one location on your computer network and share them with all your users. In either case, project templates must be stored in the Templates Folder selected in or the Program Settings Dialog. They can also be stored in a sub-folder.

The project templates you create (as the PGSuper Administrator) are to be shared amongst the users in your organization. Templates should be stored in a directory on a network server that all PGSuper users can access. By storing project templates on a network server, users automatically have access to the same information and to any updates that may occur over time.

Tip: You can customize the PGSuper installation so the Templates Folder is automatically set for your users. See Chapter 1 of the PGSuper Administrator's Guide.

Organizing Your Templates

You can organize your templates by storing them in sub-folders. Typically, templates provided with PGSuper are organized by girder type. Other organizational schemes might include geographical region, span lengths, or state standards.

All of the templates directly in the Template Folder are presented in the General tab of the New Projects dialog. Each of the sub-folders corresponds to a tab in the New Projects dialog. Deploying the Templates

To deploy your PGSuper Project Templates:

  1. Create a root folder and sub-folders that represent your organization scheme

  2. Using PGSuper, and the procedure outlined above, create your project templates.

  3. Save the templates into the project template folders

  4. If you have an installed user base, tell them the location of the project templates and how to customize PGSuper to find the folder.
