Introduction

In this chapter you will learn how to install PGSuper. PGSuper can be installed as a stand-alone application or as an application that shares resources with users on a network. The various installation options will be discussed in detail.

Quick Start

If you just want to install the darn program now, run the installer and accept the default options. The program will be installed on your computer along with a non-shared version of the Project Templates and PGSuper Master Library.

Tip: If you are evaluating PGSuper, just run the installer and accept the default options. This will get you running in stand-alone mode. If you decide to adopt PGSuper as a tool in your office, it will be wise to read this Administrator's Guide carefully so you can make the best use PGSuper's workgroup features.

Definitions

PGSuper Administrator

A person who is responsible for maintaining PGSuper for an entire office. Responsibilities include maintaining project templates and the master library.

Project Template

A file that contains seed data for a PGSuper project. Seed data includes bridge configuration, criteria, material properties etc. Project templates simplify the creation of PGSuper projects

PGSuper Master Library

A file that contains frequently used and sometimes very complex data. Users can simply refer to library entries, such as girder definitions, by name

Client

Someone who uses PGSuper

PGSuper Components

The PGSuper installation consists of four components that will be installed. The components are:

PGSuper Program

This is the program

 

PGSuper Project Templates

These are the default project templates that come with PGSuper.

PGSuper Master Library

This is the default library that comes with PGSuper.

PGSuper Administrator Tools

These are the tools used by PGSuper Administrators to manage a PGSuper installation.

Installing PGSuper

Before installing PGSuper, remove any previous versions. If you have PGSplice installed, remove this software as well and re-install the most current version.

To install PGSuper simply double click on the PGSuper MSI file. This will install the software for Stand-Alone use.

Using Microsoft's ActiveDirectory technology, you can deploy PGSuper to an entire office with group policy objects.

You can customize an installation by using installer transforms.

Note: Installing PGSuper on Windows NT/XP may require Administrator Rights (this is different than being a PGSuper Administrator). However, you do not need to have Administrator Rights to run the program.

Using PGSuper Efficiently

PGSuper is designed to be a highly efficient, and highly configurable, production precast girder design tool. We recognize that most organizations that design precast girder bridges use a high level of standardization in order to maximize efficiently and quality control. To a software engineer, "standardization" means redundancy of data and rigorous implementation of in-house processes. Hence, PGSuper is designed to minimize input of redundant data and to focus only on the input parameters that typically vary from design project to project.

Project libraries and template files are used to support design standardization and to minimize the amount of input data required to perform a design. In fact, PGSuper does not have any default girders, design criteria, or other settings. All of this "default" information is stored in the Master Library file and Templates.

Hopefully, the above paragraphs illustrate that the best way to use PGSuper is to create at least one carefully controlled shared master library to eliminate redundant data input (while increasing quality and ease of use). And, to develop a sensible collection of shared Template files to help your designers create new projects that are already close to the current design goals.

Configuring PGSuper for Shared Use

In a typical office situation, you will want all of your engineers using the same Templates and Master Library File. This will ensure that everyone is using the same project criteria, girder definitions, and other defaults.

To share the Project Templates and Master Library, copy them to shared directory on a easily accessible server. Then, configure each user's Program Settings to use these shared resources.
