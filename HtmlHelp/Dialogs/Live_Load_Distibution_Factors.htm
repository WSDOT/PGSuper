Live Load Distribution Factors

This project-specific dialog allows you to chose whether live load distribution factors are to be computed using the method specified in the current Project Criteria library entry, input manually, or computed using only the lever rule. Options are also available to override safeguards when parameters lie outside of the ranges of applicability specified in the AASHTO LRFD Bridge Design Specifications.

Compute Live Load Distribution Factors in Accordance With the Currently Selected Project Criteria

When this option is selected, distribution factors are computed using the methods specified on the Bridge Site 3 tab of the currently selected Project Criteria Library entry. The Ranges Of Applicability (ROA) for use of the distribution factor equations, are girder-type dependent and defined in section 4.6.2.2 of the LRFD Specifications. By default, PGSuper will refuse to analyze a structure where any range of applicability rule is broken. However, there may be cases when you want to override this behavior, and options are given below:

When a Parameter is Outside of the ROA:

Halt the Analysis And Issue a Warning

This is the default behavior. PGSuper will not complete the analysis unless you override this.

Ignore the ROA Requirements and Perform the Analysis Using the Equations

The LRFD equations will be used for computation of distribution factors regardless of their applicability.

Note: Ignoring the range of applicability is generally NOT RECOMMENDED, and should only be selected for special structures where you need to control the computation of distribution factors. Checks using hand calculations, or other methods, should always be done to insure that the factors are computed as you intended.

Use the Lever Rule Only for Cases Outside of the ROA Requirements

The LRFD equations will be used for computation of distribution factors where parameters are inside of the Range of Applicability. However, the lever rule will be used for cases outside the ROA for the equation. This option may seem safer than the previous option, however, the same amount of caution should be applied when using it.

Compute All Live Load Distribution Factors Using the Lever Rule

When this option is selected, all distribution factors are computed using the Lever Rule. Note that the LRFD skew factor equations are still applied, however, any ROA requirements for the skew equations are ignored. Again, use caution when using this option.

Directly Input All Live Load Distribution Factors

When this option is selected, use the input grid to define distribution factors directly for each girder and bearing location in the structure.

Using the User-Input Grid

There are two User-Input Factors grids; one for inputting factors for individual girders; and another for inputting factors at pier bearings. Toggle between the two grids by selecting the "Girder Factors" and "Abutment/Pier Factors" radio buttons.

The tab control at the bottom of the input grid is used to select which span(s) (or pier(s)) are to be edited. In the same manner as Excel, multiple tabs can be selected using Ctrl-Click and Shift-Click. When multiple tabs are selected, grid input operations are performed on all selected spans or piers.

Data can be input into the grid by clicking the mouse on individual grid cells. The arrow keys can also be used to navigate between cells. The grids work much like Microsoft Excel. In fact, you can copy and paste data to/from Excel as long as the selected ranges are the same size.

Selecting Ranges In the Grids

Copy and paste operations are typically done on a grid by first selecting a range of cells. There are several ways to select multiple grid cells (ranges):

  • Click and drag the mouse anywhere on the grid

  • Hold the Shift button while using the arrow keys

  • Click on header rows to select an entire row

  • Click on header columns to select an entire column

  • Click the top-left header box to select the entire grid

  • Hold the Ctrl key while selecting to select multiple ranges (note this only may be used when copying data from the grid)

Copying and Pasting

Copying and Pasting can be performed from the keyboard by using the following Windows standard key combinations:

  • Ctrl-C to copy selected data from the grid to the Windows Clipboard

  • Ctrl-V to paste data from the Clipboard to the selected cells

  • Ctrl-X to cut data from the grid and copy it to the Windows Clipboard

The mouse may also be used to copy and paste by clicking on the right mouse button over selected cells. This will bring up a context menu with the options "Copy" or "Paste".

Selecting Multiple Tabs On the Grid

Select multiple grid tabs by holding the Ctrl or Shift button while clicking the left mouse button on the tabs. This allows data input onto multiple grids simultaneously in the same manner as the "Drill Down" feature in Excel.

When multiple tabs are selected, any input operations (typing, and pasting) are performed on all selected grids.

Fill With Computed Values

There may be occasions when you wish to use some of the live load distribution factors computed by PGSuper and manually input others. The Fill with computed Values" option is for this scenario.

Clicking on this button will bring up the Fill Live Load Distribution Factors With Computed Values dialog box. The dialog provides several ways to fill the user-input grid with values computed by PGSuper.

Tip: Fill With Computed Values is a one-time operation. Values will NOT be automatically updated if bridge data is later modified.
