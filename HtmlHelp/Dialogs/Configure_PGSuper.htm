Program Settings - Configure PGSuper

This dialog is automatically presented the first time you run PGSuper after it is installed, or you can access if from the "File | Program Settings" menu. There are two steps to configuring PGSuper; supplying user information and selecting templates and libraries.

Libraries and Templates

PGSuper is designed to be a highly efficient, and highly configurable, production girder design tool. Project libraries allow you to define design settings and save them into a sharable file. Template files provide a starting place on which to base new projects. Together, these features support design standardization and to minimize the amount of input data required to perform a design. In fact, PGSuper does not have any default girders, design criteria, or other settings. All of this "default" information is stored in the Master Library file and Templates.

Note: PGSuper must be configured to use a Master Library and Templates.

A Note For First-Time PGSuper Users

If you are a first time user, or are trying to use PGSuper for an agency that does not publish standard PGSuper libraries, you can start by using the generic library and templates that were installed with PGSuper. These will get you up and running quickly and do not require Internet access. PGSuper also comes preconfigured to access the Internet servers at WSDOT and TxDOT which contain preconfigured standard templates and libraries for those agencies.

However, if you decide to use PGSuper as your production design tool, it is wise to create your own Master Library files with girder shapes, connections, and project design criteria for your situation. Read the Administrator's Guide for detailed instructions for setting up a production environment in your office..

Standardized Libraries and Templates from Publishers

If you are working for a DOT that has fully adopted PGSuper, such as WSDOT or TxDOT, you can download and synchronize with libraries and templates that are published on the Internet. The WSDOT server also has an option to use standard AASHTO options and AASHTO girders. PGSuper is open, so any organization is free to post libraries and templates either on the Internet or a local network. For example, www.pgsuper.com has a growing number of library/template configurations for your use.

PGSuper Libraries and Templates Server

Select the name of a PGSuper Catalog Server. PGSuper will access this server and provide access to its Master Library and Template file publishers.

Press the Add/Edit button to manage the PGSuper Catalog Servers. A PGSuper Catalog Server is defined by a name, type, and a location. Three types are supported: FTP, HTTP, and File System. The location must be a valid of the correct type  that permits anonymous read access. An example of a PGSuper Catalog Server definition is shown below:

User Information

When creating a new PGSuper project, one of the first things that you will do is supply some basic information about the project. These properties are used to describe a bridge project and to identify the engineer performing the work. Your personal information will be the same for every project. So you don't have to enter your personal information every time you create a new project, you can specify that information in the User Information fields.

Enter your name and company in the input fields. This information will be put onto reports and other output created by PGSuper.

Update Frequency

Select the frequency at which PGSuper will look for updates to the currently selected. Master Library and Template files. This option is not available for the generic files option.

Tip: Once you've selected a Publisher, you aren't "locked in". You can change the publisher at any time, and the program will still run if you disconnect your computer from the Internet. See the Administrator's Guide for more information.
