Program Settings - Configure PGSuper

This dialog is automatically presented the first time you run PGSuper after it is installed, or you can access if from the "File | Program Settings" menu. There are two steps to configuring PGSuper; supplying user information and selecting templates and libraries.

Libraries and Templates

PGSuper is designed to be a highly efficient, and highly configurable, production girder design tool. Project libraries and template files are used to support design standardization and to minimize the amount of input data required to perform a design. In fact, PGSuper does not have any default girders, design criteria, or other settings. All of this "default" information is stored in the Master Library file and Workgroup Templates.

Note: PGSuper must be configured to use a Master Library and Templates.

A Note For First-Time PGSuper Users

If you are a first time user, or are trying to use PGSuper for an agency that does not publish standard PGSuper libraries, you can start by using the generic library and templates that were installed with PGSuper. These will get you up and running quickly and do not require Internet access.  

However, if you decide to use PGSuper as your production design tool, it is wise to create your own Master Library files with girder shapes, connections, and Project Criteria for your situation. Read the Administrator's Guide for detailed instructions for setting up a production environment in your office..

Standardized Libraries and Templates from Publishers

If you are working for a DOT that has fully adopted PGSuper, such as WSDOT or TxDOT, you can download and synchronize with libraries and templates that are published on the Internet. There is also an option to use standard AASHTO options and AASHTO girders. Select a publisher to configure PGSuper to keep up to date with the latest libraries and templates for that publisher.

PGSuper Catalog Server

Select the name of a PGSuper Catalog Server. PGSuper will access this server and provide you access to the Master Library and Workgroup Template files.

Press the Add/Edit button to manage the PGSuper Catalog Servers. A PGSuper Catalog Server is defined by a name and a location. The location must be an FTP server that permits anonymous access. An example of a PGSuper Catalog Server definition is shown below:

User Information

When creating a new PGSuper project, one of the first things that you will do is supply some basic information about the project. These properties are used to describe a bridge project and to identify the engineer performing the work. Your personal information will be the same for every project. So you don't have to enter your personal information every time you create a new project, you can specify that information in the User Information fields.

Enter your name and company in the input fields. This information will be put onto reports and other output created by PGSuper.

User Templates Folder

Enter the location of your personal templates in the User Templates field. The templates stored in this folder and the folders below it will be displayed in the "File | New" dialog.

The table below shows the syntax for valid template folder paths:

Folder in a local hard disk or mapped network drive

M:\MyPGSuperTemplates\

Network folder

\\myserver\MyPGSuperTemplates\

See Chapter 10 of the PGSuper User's Guide for more information

Update Frequency

Select the frequency at which PGSuper will look for updates to the Master Library and Workgroup Template files. This option is only applicable if you've selected a Libraries and Templates option that synchronizes with published files.

NOTE: PGSuper must have an Internet connection to automatically update the library and templates. However, you do not need an active Internet connection to use PGSuper.

Tip: Once you've selected a Publisher, you aren't "locked in". You can change the publisher at any time, and the program will still run if you disconnect your compute from the Internet. See the Administrator's Guide for more information.

You can also use and synchronize with libraries and templates on your local network. Contact your PGSuper Administrator for the location of the Master Library and Workgroup Templates
