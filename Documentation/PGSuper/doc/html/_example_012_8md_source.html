3 Creating the Project
4 --------------------
5 Create a new PGSuper project file based on the W42G template. The steps are the same as for Example Bridge 1.
6 
7 Editing the Bridge Model
8 -------------------------
9 We will be using different editing techniques to create this bridge model. The techniques you've seen in previous examples will work just fine, we simply want to show you some of the other editing features of the software.
10 
11 Let's start by editing the alignment.
12 
13 1) Select *Edit > Alignment...*
14 
15 Input the alignment, profile, and superelevations as shown
16 
17 ![](Tutorial_Advanced_Example_2_Alignment_Horizontal.png)
18 ![](Tutorial_Advanced_Example_2_Alignment_Profile.png)
19 ![](Tutorial_Advanced_Example_2_Alignment_Superelevation.png)
20 
21 2) General Bridge Settings
22 
23 We need to set the general bridge information. Specifically, this bridge uses a different girder spacing in each span. A quick way to get at the Bridge Description information is to double click the left mouse button while the mouse pointer is over the whitespace Bridge Model View.
24 
25 Set the number of girders to 7 and remove the check mark for girder spacing in all spans
26 
27 ![](Tutorial_Advanced_Example_2_Bridge_Description_General.png)
28 
29 3) Add spans
30 
31 This is a three span bridge, so we need to add two more spans. We could use the framing grid on the Framing tab, but we've done that before. Let's learn a new technique.
32 
33 Right click the mouse on Abutment 2 in the Bridge Model View. Select Insert from the pop-up menu. An Insert Span dialog is presented. Press [OK] to create a span.
34 
35 Repeat this process to create the third span
36 
37 4) Define Span Lengths
38 
39 We are going to input the span lengths using a feature known as In-place Editing. Left button click the mouse on the length of Span 1 in the Bridge View. An edit box should appear. Input the span length of 43.74ft
40 
41 ![](Tutorial_Advanced_Example_2_Inplace_Edit.png)
42 
43 Repeat this process for Spans 2 and 3. The span lengths are 68.48 ft and 63.93 ft respectively.
44 
45 5) Position the Bridge along the alignment
46 
47 The spans have the correct length along the alignment, however the bridge is in the wrong position. We can move the entire bridge by setting the location of one of the piers. Double-click on Abutment 1 to bring up the Abutment Details window.
48 
49 Enter the station of Abutment 1. Since we want to move the bridge, select the Move bridge retaining all span lengths option.
50 
51 > TIP: The length of adjacent spans can be changed by moving a pier.
52 
53 Since we are editing the details of Abutment 1, we may as well set the pier orientation.
54 
55 ![](Tutorial_Advanced_Example_2_Abutment_1.png)
56 
57 Before closing this window, let's set the girder spacing at Abutment 1. Select the Girder Spacing tab.
58 
59 Set the girder spacing to Measure at and along the centerline bearing.
60 
61 Place the mouse arrow over the column of the spacing grid labeled A-G (ft). The cursor will change to a black down arrow.
62 
63 Right click on the gird and select Expand. This will expand the grid, allowing for spacing input between every girder.
64 
65 Using the left mouse button, click on the column labeled B-C (ft) and drag the mouse to the column labeled E-F (ft). This will select four columns as shown.
66 
67 ![](Tutorial_Advanced_Example_2_Abutment_1_Spacing_Expanded.png)
68 
69 Right click on the selected group of columns and select Join. This will join the four columns into one labeled B-F (ft). Enter the girder spacing.
70 
71 ![](Tutorial_Advanced_Example_2_Abutment_1_Spacing_Joined.png)
72 
73 Repeat this process for Pier 2, 3, and Abutment 4. Note that girder spacing is measured differently at the interior piers.
74 
75 The spacing on both sides of the interior piers is the same. To simplify input, define the spacing on the back side of the pier and press [Copy to Ahead Side of Pier].
76 
77 ![](Tutorial_Advanced_Example_2_Pier_2_Spacing.png)
78 
79 Press [OK] to apply all these changes to the bridge model. The resulting model is shown in the Bridge Model View
80 
81 ![](Tutorial_Advanced_Example_2_Bridge_Model_View.png)
