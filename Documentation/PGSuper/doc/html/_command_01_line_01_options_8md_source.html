1 Command Line Options {#ug_command_line_options}
2 ==============================================
3 Command line options are parameters that are passed to the program when execution begins. The options alter the normal behavior of the software or provide some other special functionality. This chapter describes the command line options.
4 
5 PGSuper Command Options
6 ------------------------
7 
8 PGSuper supports the following command options
9 
10 Option | Description
11 -------|--------------
12 /Configuration=<i>ServerName</i>:<i>PublisherName</i> | Sets the application configuration.
13 /TestR | Generates NCHRP 12-50 test results for all problem domains
14 /Test<i>n</i> | Generates NCHRP 12-50 test results for a specified problem domain. Substitute the problem domain ID for <i>n</i>.
15 
16 TxDOT Data Exporter
17 -------------------
18 
19 > NOTE: You must select the TxDOT components at installation time in order to use this feature.
20 
21 When the TxDOT data export options are given on the command line, PGSuper will create an output file that contains the TxDOT CAD data. The output generated when using the "Tx" options is very similar to that which is generated by PSTRS14. This feature can be used for batch processing.
22 
23 ### Command Line Options ("Tx" Options) ###
24 To run PGSuper with the Tx options, use a command with the following format:
25 
26  BridgeLink.exe /PGSuper /Option input_file.pgs Output_file <span> <girder>
27 
28 Where:
29 * /PGSuper is the BridgeLink command option that directs the remainder of the command line to PGSuper for processing
30 * Option (case in-sensitive) can be:
31  - TxA, Run analysis of current data, output standard CAD format
32  - TxAx, Run analysis of current data, output extended testing format
33  - TxD, Run flexure design, output standard CAD format
34  - TxDx, Run flexure design, output extended testing format
35  - TxDS, Run flexure and shear design, output extended testing format
36  - TxAo, Same as TxA but will overwrite output file instead of appending
37  - TxAxo, Same as TxAx but will overwrite output file instead of appending
38  - TxDo, Same as TxD but will overwrite output file instead of appending
39  - TxDxo, Same as TxDx but will overwrite output file instead of appending
40 * Input_file.pgs is the input file to be processed
41 * Output_file is the name of the output file. Existing files will be overwritten.
42 * <span> is a single integer (e.g., 1, 2, ...) defining the span where the girder is, or "All" to run all spans.
43 * <girder> is a single character (e.g., a, B, f, ...) defining the girder number, or "All" to run all girders in a span (or all girders in all spans if span option is also "All").
44 
45 #### Notes ####
46 If an error occurs while running PGSuper, a .err file (e.g., "Route66.err") attempting to explain the problem will be created. Partial data will likely be in the output file.
47 
48 Library Entry conflicts will cause processing to cease. You need to make sure that there are no library conflicts in the input file before running them from the command line.
49 
50 > NOTE The file format for this command has been defined by TxDOT and can be changed by them without notice.
51 
52 ### Examples ###
53 An example of a command line to run a design for span 1, girder C using input file "Route66.pgs" and creating an extended output file "Route66.txt" is as follows:
54 
55  BridgeLink.exe /PGSuper /TxDx Route66.pgs Route66.txt 1 C
56 
57 Here is another example to run an analysis using standard output format for all beams in Span 2:
58 
59  BridgeLink.exe /PGSuper /TxA Route66.pgs Route66.txt 2 ALL
60 
61 ### Output File Format ###
62 Output files are generated in "standard" or "enhanced" format. These are space-delimited text files (fields are separated by a single blank space). The data formats for draped and debonded strand layouts are different by nature. Refer to the TxDOT IGND and UBND girder design standard plan sheets for more information about data and formatting.
63 
64 > TIP: The extended version of the file contains extra data and a table header that describes the content.
